How selected text inside the JSON tree will look. The highlighted spans simulate a text selection drag.

Option A — Teal Accent Recommended
JSON Output ⟨ ⟩ ⛶
{
"name": "my-server",
"connectionId": "abc-123-def",
"port": 8080,
"active": true,
"tags": ["prod", "v2"]
}
Teal `rgba(56,189,248, 0.22)` + teal outline Matches the app accent colour. Subtle, readable, doesn't compete with the syntax highlights. Search input gets a teal border glow to match.

Gray-blue `#2d3748` + mid-gray outline The most conservative option. Replaces the jarring white with a dark slate, so text always remains readable. Search input gets a plain gray border — the "outer light gray border" you suggested.
What changes
1. Text selection colour (::selection) inside the JSON tree
2. Search input field background + border (replacing PrimeVue white default)
3. Consistent across inline view + fullscreen dialog
My recommendation
Option A — teal is already the app's accent, so selection highlights feel native rather than decorative. The search input glow also reinforces which field is active.

Drag-select the text in any card above to preview the browser's actual ::selection colour — Note: mockup uses outline on a span to simulate selection since real ::selection can't be conditionally applied in HTML.
